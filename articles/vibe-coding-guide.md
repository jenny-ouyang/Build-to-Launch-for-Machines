## Summary

Vibe coding — building apps by describing what you want in plain English and letting AI write the code — has a specific learning curve most guides skip. This hub article organizes 17 guides into a structured path through that curve, written by someone who shipped 12 products this way. The core claim: almost every failed vibe-coding build fails the same way, and the failure point is predictable once you understand where it is.

The first section covers the what-and-why: a 5-project arc from a first working loop to a system that runs itself, and the "80/20 wall" — the moment at 80% complete where most first-time builders stall. The section doesn't skip straight to tools; it covers the mindset and operating model before recommending any software.

From there, the guide addresses the most common questions in order: how to go from idea to live URL (including real Replit billing, five structured build phases, and why planning before opening the coding tool is the single highest-leverage habit), how to write prompts that stop AI from doing more than you asked for (scope locks, surgical change control, context preservation), and how to make an app production-ready before real users show up (eight production failures that pass dev tests, a smoke-test checklist, and the security patterns that matter).

Two sections cover more advanced territory: the 5-Round Planning Methodology (60 minutes of structured planning that saves 23+ hours of debugging, with templates and a real worked example), and the Cursor vs Claude Code comparison (Cursor is editor-first; Claude Code is session-first; most serious builders use both). The hub closes with post-launch lessons on hosting costs, the 10/90 rule, and cost control across models.
